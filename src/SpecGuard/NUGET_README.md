# SpecGuard

Drop-in ASP.NET Core middleware that validates incoming API requests against your OpenAPI specification.

```csharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddOpenApi();
builder.Services.AddSpecGuard();

var app = builder.Build();

app.UseSpecGuard();
app.MapEndpoints();
app.MapOpenApi();

app.Run();
```

As long as the OpenAPI specification generated by your API is complete and correct, SpecGuard just works.

## Why

SpecGuard validates JSON requests against the OpenAPI specification published by your API at runtime, producing error messages that refer to the actual JSON submitted by the client — with line/column numbers, JSON paths, and meaningful descriptions. Parameters outside the request body are validated too, with consistent error messages.

This prevents drift between the published specification and your validation logic, and helps you maintain a good OpenAPI specification.

## Requirements

- `net10.0`
- ASP.NET Core's built-in OpenAPI stack (`Microsoft.AspNetCore.OpenApi`)
