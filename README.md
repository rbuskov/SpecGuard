# SpecGuard

Drop-in ASP.NET middleware that validates incoming API requests against your OpenAPI specification:

```csharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddOpenApi();
builder.Services.AddSpecGuard(); // Register SpecGuard types

var app = builder.Build();

app.UseSpecGuard();              // Configure SpecGuard middleware
app.MapEndpoints();
app.MapOpenApi();

app.Run();
```
As long as the OpenAPI specification generated by your API is complete and correct, SpecGuard just works. See the
[usage documentation](./USAGE.md) for details.

## Why

In the .NET world, the standard approach is to perform validation on server-side CLR models after JSON requests have 
been deserialized. This often causes a mismatch between the actual JSON request and error messages that refer to 
server-side models.

SpecGuard works at the JSON level to produce validation error messages that consistently refer to the actual JSON 
submitted by the client, with line/column numbers, JSON paths and meaningful descriptions. 

Validation is based on the OpenAPI specification published by your API at runtime, and works by converting the 
specification into JSON schemas that are used to validate requests before deserialization. SpecGuard also validates 
parameters that are not part of the request body and produces consistent error messages for those as well.

Apart from the obvious benefits of validation, this approach also prevents drift between the published specification 
and your actual validation logic, and it helps you to create and maintain a good OpenAPI specification for your API.

## Samples

The solution ships with these sample API projects:

- [`SpecGuard.MuseumApi`](./src/SpecGuard.MuseumApi) — a rich domain (museum hours, special events, tickets) demonstrating validation across multiple resources.
- [`SpecGuard.TicTacToeApi`](./src/SpecGuard.TicTacToeApi) — a small API that models a game of Tic-Tac-Toe.

## Installation

SpecGuard is available on [NuGet](https://www.nuget.org/packages/SpecGuard). While in beta, install with the `--prerelease` flag:

```bash
dotnet add package SpecGuard --prerelease
```

Or add the package reference directly to your `.csproj`:

```xml
<PackageReference Include="SpecGuard" Version="0.9.0-beta.2" />
```

SpecGuard targets `net10.0` and depends on ASP.NET Core's built-in OpenAPI stack (`Microsoft.AspNetCore.OpenApi`).
